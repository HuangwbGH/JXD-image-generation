# 实施里程碑

## 第 1 阶段：项目脚手架

交付物：

- Next.js + TypeScript 项目。
- React、CSS、lucide-react。
- 登录页、自助注册页、基础会话能力。
- 基础布局：左侧控制区、右侧预览区。
- `.env.example`。

验收标准：

- 本地 `npm run dev` 可启动。
- 用户未登录时进入登录页。
- 自助注册后需要管理员审核才能登录。
- 注册字段用户名、密码、展示名、部门均必填。
- 首页展示和参考图一致的工具台结构。

## 第 2 阶段：Gemini 生成闭环

交付物：

- `/api/generate`。
- `lib/gemini/generate.ts`。
- 文生图和图生图请求。
- 图片保存与静态访问。

验收标准：

- 为部门填入 API Key 后可以生成图片。
- 普通用户生成时自动使用账号所属部门的 API Key。
- 上传图片后可以根据提示词编辑图片。
- 生成失败时前端展示可理解的错误。

## 第 3 阶段：配置中心

交付物：

- 设置页。
- 用户管理：研发一部用户、研发二部用户、管理员。
- 自助注册审核。
- 部门 API Key、模型、程序级代理、上传限制配置。
- `/api/settings/test` 连通性测试。

验收标准：

- 可以在页面按部门配置或修改 API Key。
- 管理员可以创建和维护用户账号。
- 管理员可以审核注册申请。
- 普通用户不能访问设置页。
- 可以启用代理 `http://192.168.1.27:20171` 后测试 Gemini 连通性。
- 代理只作用于本程序访问 Gemini 的请求。
- API Key 不出现在浏览器响应明文中。

## 第 4 阶段：记录与体验完善

交付物：

- 生成记录表。
- 下载、复制、重新生成、继续编辑。
- 手动删除生成记录和图片。
- 最近生成历史。
- 全局并发限制和请求超时。

验收标准：

- 生成结果可追踪。
- 普通用户只能看到自己的生成记录。
- 管理员可以看到所有生成记录。
- 普通用户只能删除自己的生成记录和图片。
- 管理员可以删除任意生成记录和图片。
- 删除采用物理删除。
- 生成图片默认永久保存。
- 多人同时使用时不会无限制打满后端。
- 常见失败有明确提示。

## 第 5 阶段：内部部署

交付物：

- Dockerfile。
- docker-compose.yml。
- 健康检查接口。
- 部署说明。

验收标准：

- 内网服务器部署成功。
- Docker 部署可在 Linux OS 上运行。
- 公司同事登录后可访问使用。
- 运维可以通过设置页更新程序级代理和部门 API Key。

## 开发原则

- 开发必须遵循 `../../.openclaw/workspace/archive/规范/2026-04-20-LLM-写代码规范.md`。
- 所有路径使用相对路径。
