# 产品与范围规划

## 目标用户

公司内部需要快速制作图片素材的同事，例如研发一部、研发二部及管理员。系统需要登录来区分用户、部门和记录归属；不做计费，不审计上传图片内容，但需要保护 API Key、代理等敏感配置。

## 功能范围

### MVP 必须实现

- 文生图：输入提示词，选择模型、比例、分辨率、输出格式后生成图片。
- 图生图：上传 1 至多张图片，输入编辑提示词，生成改图结果。
- 用户登录：本地账号密码登录后才能使用生图、记录和设置功能。
- 自助注册：普通用户可自助注册，注册字段为用户名、密码、展示名、部门，全部必填；注册后等待管理员审核。
- 用户与部门：普通用户归属于研发一部或研发二部，生成时自动使用所属部门 API Key。
- 管理员：可以维护用户、部门 API Key、代理、模型和上传限制，并查看全部生成记录。
- 图片上传：支持 png、jpg、jpeg、webp，拖拽上传，显示数量和大小限制。
- 结果预览：右侧大画布展示生成状态、生成结果、模型返回文本、错误信息。
- 结果操作：下载 PNG/JPG、复制图片、重新生成、查看原始提示词。
- 配置中心：按部门配置 Gemini API Key，配置模型、程序级 HTTP/HTTPS 代理、上传上限、默认输出参数。
- 使用记录：保存用户、部门、生成时间、输入参数、参考图数量、输出路径、错误摘要；普通用户只看自己的记录，管理员看全部记录；页面支持列表模式和详情模式。
- 系统日志：管理员查看登录、注册、审核、设置、生成请求、生成响应、错误和删除日志。

### 不做的功能

- 订阅、计费、积分包、支付。
- 公共作品广场、公开分享页。
- 视频生成。
- 多供应商模型市场。
- 部门级生成次数限制。
- 部门级并发限制。

### 可选增强

- 提示词模板库：电商主图、海报、头像、风格迁移、局部修改。
- 批量生成：同一提示词生成多张候选图。
- 多轮编辑：基于上一次结果继续调整。
- 简单敏感词与版权风险提示。
- SSO 或企业通讯录集成。
- 部门级用量统计。

## 页面结构

### 主工具页

左侧为参数面板：

- 模式切换：`图像编辑`、`文生图`。
- 当前用户：显示登录用户、角色和所属部门。
- AI 模型：默认 `gemini-3.1-flash-image-preview`，可切换 `gemini-3-pro-image-preview` 和 `gemini-2.5-flash-image`。
- 图片上传：图生图模式显示，文生图模式可折叠隐藏。
- 提示词：最大长度建议 5000 字符。
- 分辨率：`1K`、`2K`、`4K`，按模型能力限制可选项。
- 输出格式：`PNG`、`JPG`。
- 纵横比：`自动`、`1:1`、`9:16`、`16:9`、`3:4`、`4:3`、`3:2`、`2:3`、`5:4`、`4:5`、`21:9`。
- 生成按钮：请求中显示进度和禁用重复提交。
- 即将发送内容：实时展示模式、模型、部门、提示词、比例、分辨率、输出格式和上传图片摘要。

右侧为工作区：

- 空状态：显示“就绪创建”。
- 生成中：显示加载状态、当前模型、耗时。
- 成功：展示图片预览、下载、复制、继续编辑。
- 失败：展示错误摘要、重试按钮、配置检查入口。

### 设置页

- 用户管理：管理员审核自助注册用户，维护用户账号、角色和所属部门。
- 部门与 Gemini API Key：由运维维护，每个部门独立 API Key，保存前可测试连通性。
- 模型配置：默认模型、允许用户选择的模型列表。
- 代理配置：启用/禁用程序级代理，HTTP 代理地址，HTTPS 代理地址。代理只作用于本程序访问 Gemini 的请求，不要求整台服务器走代理。
- 上传限制：单图大小、最多图片数、允许 MIME 类型。
- 存储配置：本地输出目录或对象存储配置。
- 系统状态：Gemini 连通性、代理连通性、输出目录可写性。

## 交互原则

- 默认打开就是生图工具，不做营销首页。
- 未登录用户进入系统时先跳转登录页。
- 内部工具优先密度和效率，避免大面积宣传内容。
- 所有敏感配置只在服务端读取或保存，前端永不接触明文 API Key。
- 错误提示要可操作，例如“当前部门 API Key 未配置”“代理不可用”“模型不支持 4K”。

## 风险与约束

- Gemini 图像 API 在不同地区可能有可用性差异，部署时需要支持公司代理。
- 公司服务器访问 Gemini 需要程序级代理，不能依赖服务器全局代理。
- 生成图片包含 Google SynthID 水印，这是模型侧行为，不应承诺去除。
- 上传图片需要用户自行确认有使用权，页面应给出简短提醒。
- 生成图片默认永久保存在公司内部服务器上。
- 支持物理删除生成图片和记录；普通用户只能删除自己的内容，管理员可以删除全部内容。
- 输出图片文件名使用 `用户ID+年月日时分秒`。
- 不限制每个部门的生成次数和并发数；仍建议保留全局并发限制，避免服务被瞬时请求打满。
- 所有路径使用相对路径，部署使用 Docker 并支持 Linux OS。
- 开发必须遵循 `../../.openclaw/workspace/archive/规范/2026-04-20-LLM-写代码规范.md`。
