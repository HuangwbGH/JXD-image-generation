## 2. 背景

公司内部运营、产品、市场、内容、设计等团队经常需要生成活动海报、社媒配图、产品概念图、风格参考图和图片二次编辑结果。现有方式依赖外部网站或个人账号，存在 API Key 管理分散、部门用量难拆分、代理配置不统一、素材沉淀困难、使用体验不稳定等问题。

本项目建设一个内部可登录访问的 Gemini 生图网站，统一接入 Gemini 图像生成能力，提供文生图和图生图两类核心能力。系统通过用户账号区分部门和记录归属，但不建设计费、积分、订阅等商业化功能。

## 3. 产品目标

### 3.1 业务目标

- 让内部同事可以在 1 个页面内完成文生图和图生图。
- 通过登录账号区分研发一部、研发二部和管理员权限。
- 统一管理不同部门的 Gemini API Key、模型、程序级代理和上传限制。
- 降低外部工具依赖，提升素材生成效率。
- 为后续模板库、批量生成、多轮编辑、使用统计预留扩展空间。

### 3.2 用户目标

- 登录后即可使用。
- 可以上传图片并用自然语言描述修改要求。
- 可以输入提示词直接生成图片。
- 可以选择常用比例、分辨率和输出格式。
- 生成失败时能看到明确原因和处理建议。

## 4. 用户角色

## 6. 核心流程

### 6.1 文生图流程

1. 用户打开网站。
2. 用户登录。
3. 系统根据账号识别用户角色和部门。
4. 用户进入首页并选择 `文生图`。
5. 输入提示词。
6. 选择模型、比例、分辨率、输出格式。
7. 点击生成。
8. 系统使用用户所属部门的 Gemini API Key 调用 Gemini。
9. 右侧展示生成状态。
10. 成功后展示图片。
11. 用户下载、复制、重新生成或继续编辑。

### 6.2 图生图流程

1. 用户打开网站。
2. 用户登录。
3. 系统根据账号识别用户角色和部门。
4. 用户进入首页并选择 `图像编辑`。
5. 上传 1 至多张图片。
6. 输入编辑要求。
7. 选择模型、比例、分辨率、输出格式。
8. 点击生成。
9. 系统使用用户所属部门的 Gemini API Key 调用 Gemini。
10. 成功后展示编辑后的图片。
11. 用户下载、重新生成或继续编辑。

### 6.3 配置流程

1. 运维或管理员打开设置页。
2. 系统校验管理员权限。
3. 审核自助注册用户。
4. 维护用户账号、角色和所属部门。
5. 维护部门列表。
6. 为研发一部、研发二部配置对应的 Gemini API Key。
7. 选择默认模型。
8. 配置是否启用程序级代理。
9. 如启用代理，填写 HTTP/HTTPS 代理地址。
10. 点击测试连通性。
11. 测试通过后保存配置。

## 8. 非功能需求

### 8.1 性能

- 首页首屏加载时间目标小于 3 秒。
- 生成请求前端超时提示不超过 120 秒。
- 图片上传和预览不应阻塞页面交互。

### 8.2 可用性

- Gemini API 不可用时给出明确错误。
- 代理不可用时给出明确错误。
- 当前部门 API Key 未配置时，普通用户提示联系管理员，管理员可进入设置页处理。
- 输出目录不可写时设置页测试应失败并展示原因。
- 登录过期时跳转登录页。

### 8.3 安全

- API Key 不暴露给浏览器。
- 用户密码或登录凭据必须加密存储，不允许明文保存。
- 普通用户不能访问管理员接口。
- 上传文件必须重命名。
- 禁止上传 SVG、HTML、脚本等非图片文件。
- 配置接口仅面向内网访问。
- 日志不得记录 API Key 明文。
- 删除图片需要同时更新生成记录状态，避免记录仍指向不存在的文件。
- 所有程序配置、存储路径和文档示例路径使用相对路径，避免绑定具体机器路径。

### 8.4 兼容性

- 桌面端 Chrome、Edge、Safari 最新两个大版本。
- MVP 不强制适配 IE 或旧版浏览器。

### 8.5 可维护性

- Gemini 调用封装在独立模块。
- 代理配置封装在独立模块，并且只作用于本程序的 Gemini 请求。
- 存储层保留从本地文件切换到对象存储的接口。

## 9. 埋点与日志

MVP 不做复杂数据分析，但后端日志需要覆盖：

- 生成任务 ID。
- 用户标识。
- 部门标识。
- 请求模式。
- 模型。
- 参考图数量。
- 输出比例。
- 分辨率。
- 成功或失败。
- 错误摘要。
- 耗时。

不记录：

- API Key 明文。
- 用户密码明文。
- 代理密码明文。
- 上传图片原始文件名中的敏感路径。
- 上传图片内容审计结果。

## 10. 验收标准

### 10.1 文生图验收

- 输入提示词后能成功生成图片。
- 可选择至少 3 种常用比例。
- 可选择 PNG/JPG 输出。
- 可下载生成结果。
- 生成失败时能看到错误原因。

### 10.2 图生图验收

- 上传 png/jpg/webp 图片后能生成编辑结果。
- 超过上传数量限制时前端阻止并提示。
- 超过文件大小限制时前后端均能拦截。
- 生成结果可继续作为参考图再次编辑。

### 10.3 配置验收

- 可以按部门配置 Gemini API Key。
- API Key 不会在接口响应中明文返回。
- 普通用户无法访问设置页。
- 管理员可以维护研发一部、研发二部用户和管理员账号。
- 管理员可以审核自助注册用户。
- 可以配置并启用代理 `http://192.168.1.27:20171`。
- 代理只作用于本程序发起的 Gemini 请求，不影响服务器其他进程。
- 可以在设置页测试 Gemini 连通性。
- 代理配置错误时能看到失败提示。

### 10.4 部署验收

- Docker Compose 可以启动服务。
- 使用 Docker 部署，并能在 Linux OS 上运行。
- 输出目录可持久化。
- 服务重启后配置仍然存在。
- 用户必须登录后才能访问系统。
- 生成图片在公司内部服务器默认永久保存。

### 10.5 记录权限验收

- 研发一部用户只能看到自己发起的生成记录。
- 研发二部用户只能看到自己发起的生成记录。
- 管理员可以看到所有用户的生成记录。
- 普通用户可以删除自己的生成记录和图片。
- 管理员可以删除任意生成记录和图片。
- 删除采用物理删除，文件和数据库记录均被移除。
- 输出文件命名符合 `用户ID+年月日时分秒` 规则。

## 13. 待确认问题

### 13.1 已确认结论

- 服务器不是固定能访问 Gemini，必须支持在程序内配置公司内部代理。
- 代理必须是程序级代理，只让本程序访问 Gemini 时走代理，不要求整个服务器走代理。
- Gemini API Key 由运维维护。
- 不同部门使用不同 Gemini API Key。
- MVP 部门为研发一部和研发二部。
- 系统需要用户登录。
- 登录方式为本地账号密码。
- 管理员账号名固定为 `admin`。
- 管理员密码哈希由运维在数据库中维护。
- 用户可以自助注册，注册后需管理员审核。
- 自助注册字段为用户名、密码、展示名、部门，全部必填。
- 管理员拒绝注册申请后，用户允许使用同一账号重新提交注册。
- 管理员可以查看全部生成记录。
