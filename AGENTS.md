## 适用范围

本文件约束本项目中的 AI 编程工具和人工开发者。执行任务时必须先读取本文件，再读取 `PRD.md`、`docs/Index.md` 和与任务相关的专题文档。

## 事实来源优先级

1. `PRD.md`
2. `docs/Milestones/`
3. `docs/Specs/`
4. `docs/ProcessTodo.md`
5. 代码、配置和测试
6. 当前对话补充说明

如果当前对话改变需求、配置、部署、接口或验收方式，必须同步更新对应文档。

## 工作闭环

- 开始前明确目标、范围、不做范围、影响面和验证方式。
- 保持小步修改，只改当前任务需要的文件。
- 修改代码后运行必要检查；无法运行时说明原因、影响和替代验证方式。
- 完成后更新 `docs/ProcessTodo.md`、相关专题文档和测试记录。

## 项目结构约束

- 根目录必须保留 `README.md`、`PRD.md`、`AGENTS.md`、`.env.example`、`Dockerfile`、`docker-compose.yml`。
- `docs/` 是项目文档主目录，入口为 `docs/Index.md`。
- 正式代码不得依赖 `webMock/`。当前项目没有 `webMock/`。
- 运行数据放在 `storage/`，生产数据不得提交到 Git。

## 编码约束

- 使用现有技术栈：Next.js App Router、React、TypeScript、SQLite、Docker。
- 不引入用户未要求的功能，不做无关重构。
- 所有可配置项必须来自 `.env`、数据库设置或设置页，不在业务代码里写死新配置。
- 项目内路径示例使用相对路径；代码中涉及运行时路径时优先读取配置。
- 普通用户权限、管理员权限、部门隔离和生成记录访问控制不得被绕过。

## 配置与密钥

- 真实 `.env`、Gemini API Key、密码和 Token 不得提交。
- 部门 Gemini API Key 只能通过管理员设置页维护，加密保存在 SQLite。
- `AUTH_SECRET` 必须使用足够长的随机字符串，并随数据库一起备份。
- 输出目录、上传目录和数据库路径必须可配置，并优先使用相对路径。

## Docker 与部署

- 项目必须保持 Docker 可构建、可在 Linux OS 运行。
- 修改端口、环境变量、数据卷或部署流程后，必须同步更新 `docs/Deployment.md` 和 `docs/Config.md`。
- 当前 `docker-compose.yml` 端口映射仍写死为 `32179:32179`，改动时应参数化并更新文档。

## 测试与验收

每次修改后按影响范围选择：

```bash
npm run lint
npm run typecheck
npm run build
docker compose config
curl -i http://127.0.0.1:32179/api/health
```

涉及 Gemini、代理、文件上传、记录删除和权限隔离时，需要补充人工验收记录到 `docs/TestReport.md`。

## 文档维护

- 所有 Markdown 文档必须有目录。
- 文档内容必须和代码、配置、数据库、部署方式保持一致。
- 不能确认的内容标记为 `待确认`，并写明需要谁提供什么。
- 废弃内容标记为 `已废弃`，并说明替代文档或原因。
- 同一事实只保留一个权威来源，其他文档摘要并链接。
